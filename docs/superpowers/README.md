# Superpowers Workflow

This project uses a lightweight superpowers structure to track evolving homework needs.

## Structure

- `docs/superpowers/specs/`: design specs for each new requirement.
- `docs/superpowers/specs/spec-template.md`: template for writing a new spec.

## How to use

1. For each new requirement, create a spec file under `docs/superpowers/specs/`.
2. Name it with date + topic, for example: `2026-04-02-parser-output-design.md`.
3. Implement only after the spec is reviewed and confirmed.

This keeps requirement changes traceable while avoiding unnecessary process overhead.
