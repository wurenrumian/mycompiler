# 实现文档

本文档描述项目各组件的详细实现。

## 组件列表

### 1. Stream（`include/Stream.h`）
- **用途**：通用流容器模板类
- **功能**：`next()`、`peek()`、`unget()` 操作
- **使用者**：Lexer 用于字符流处理

### 2. Token（`include/Token.h`、`src/Token.cpp`）
- **用途**：词法单元表示
- **功能**：Token 类型、词素、源码位置追踪
- **使用者**：Lexer 生成 token

### 3. Lexer（`include/Lexer.h`、`src/Lexer.cpp`）
- **用途**：CMM 语言词法分析器
- **功能**：
  - 保留字（19 个关键字）
  - 运算符（13 种）
  - 界符（8 种）
  - 整数常量（十进制、十六进制）
  - 标识符和字符串常量
  - 位置追踪（行号、列号）
  - 单行/多行注释跳过

### 4. Parser（`src/sysy.y`）
- **用途**：CMM 语言语法分析器（Bison/GLR）
- **功能**：
  - GLR 解析器处理歧义文法
  - 完整 SysY 文法覆盖
  - 通过 `Ast.h` 工具构建 AST
  - 可配置的输出控制

### 5. AST 工具（`include/Ast.h`）
- **用途**：AST 节点管理
- **功能**：AST 项栈、节点创建、树操作

## 架构

```
源代码
    │
    ▼
┌─────────────────┐
│   Lexer          │  Stream<char> → Token 流
│ (Homework 1)    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│   Parser        │  Token 流 → AST / IR
│ (Homework 2/3)  │
└─────────────────┘
```

## 关键设计决策

1. **Stream 模板**：`Stream<T>` 通用模板支持灵活的输入处理
2. **GLR 解析器**：自然处理左递归文法
3. **Token-Lexer 分离**：词法分析与句法分析间清晰的接口
4. **输出控制**：运行时切换 parser 输出（用于调试）
