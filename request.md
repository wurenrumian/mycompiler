在本次的作业中，我们需要将C语言代码，通过编译器，生成其LLVM IR的中间代码。生成 LLVM IR 是因为它是一种与机器无关的中间表示，既能解耦编译器前后端降低开发复杂度，又能直接利用 LLVM 成熟的优化工具链为后续代码优化（如常量传播、死代码消除等）提供统一且可扩展的基础设施。
简单来说，实现LLVM IR的生成，再实现MIPS目标代码的生成就很容易；同时，中间代码相比于MIPS目标代码能够更好的帮助我们完成最后的代码优化。

【问题描述】
请在词法分析、语法分析及错误处理作业的基础上，为编译器实现语义分析、代码生成功能。本次实验的目标是生成 LLVM IR 中间代码。输入输出及处理要求如下：

（1）需根据文法规则及语义约定，采用自顶向下的语法制导翻译技术，进行语义分析并生成 LLVM IR 目标代码。

（2）对于生成 LLVM IR 的编译器，约定如下：

完成编译器，将源文件（统一命名为 testfile.txt）编译生成 LLVM IR 代码，并必须输出到当前运行目录下的文件（统一命名为 output.ll），具体要求包括：

a）强烈建议同学们根据课程要求的 LLVM IR 子集规范设计中间代码数据结构，并设计实现输出中间代码的有关接口。

b）后续的作业如果参加竞速排序，请提前预留代码优化有关的接口，并设计方便切换开启/关闭优化的模式。

c）自行调试时，可使用本地安装的 LLVM 工具链中的 lli（LLVM Interpreter）命令解释运行代码。提交到平台的编译器只需要能按统一的要求生成合规的 output.ll 代码文件即可。

d）此类提交的编译器请仅读取 testfile.txt 文件并生成相应的 LLVM IR 代码，编译器自身不要读入标准输入中的内容。
【输入形式】testfile.txt 为符合文法要求的测试程序。另外可能存在来自于标准输入的数据（这些数据会在评测机用 lli 执行你生成的代码时作为标准输入传入）。

【输出形式】按如上要求将中间代码生成结果输出至 output.ll 中。

【样例输入】testfile.txt 如下：

int test;
int main(){
    printf("Hello World\n");
    test = getint();
    printf("%d",test);
    return 0;
}
【样例输出】编译器生成的中间代码 output.ll，此处省略。

【评分标准】评测机会利用 lli 读取并解释运行你生成的 output.ll 中间代码。

标准输入 input.txt 如下（此处的输入是生成的 LLVM IR 中间代码运行时接收的输入，同学们的编译器运行时无需考虑且不要读取此输入）：
1906


lli 执行 output.ll 得到的输出 output.txt 如下：
Hello World
1906
按生成代码的运行结果与预期结果不一致的行数扣分，每项扣10%。

【特别说明】

（1）本次作业测试程序中包含全部文法。
（2）本次作业仅用正确的测试程序进行考核。
（3）在本次作业中，同学们需要经常利用 lli 解释器检测自己生成的 LLVM IR 代码的正确性。（建议同学们在本地配置好 LLVM 运行环境，可使用 lli output.ll < input.txt 的方式在本地模拟评测机测试流程）。
（4）注意：测试程序（如 main 函数）的返回值将作为 lli 进程的 exit code 并被评测机接收，请确保生成的 IR 正确实现了题目测例要求的相应返回值计算。

【开发语言及环境】

平台采用的评测步骤为：首先编译你的编译器代码，然后运行你的编译器生成 output.ll，最后调用系统环境的 lli 执行该 IR 文件。
用 C/C++/JAVA 实现，机房安装的 C/C++ 开发环境是 CLion 2018.3.4 和 CodeBlocks 20.03；Java 的开发环境为 IDEA 2018.3.6 社区版。平台已配置好原生 LLVM 工具链，支持 C++11 标准。评测机所采用的编译学生代码的版本是：C/C++ gcc 8.1.0，Java jdk 1.8。
